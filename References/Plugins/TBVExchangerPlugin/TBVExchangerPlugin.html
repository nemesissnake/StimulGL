Introduction

    This Plug-in provides a direct connection to exchange different kind of data between Turbo-BrainVoyager and StimulGL. Turbo-BrainVoyager (TBV) is a highly optimized, easy to use software package for the real-time analysis and dynamic visualization of functional magnetic resonance imaging data sets. Most of the Plug-in functions used in TBV are also available in this plug-in. The data access is realized using the direct connection-oriented transmission control protocol (TCP). This allows a reliable and fast transmission of the data.

    Plug-in access

    An object of the TBV Exchanger Plug-in can be created using the following script command:

    var ObjectName = new TBVExchanger();

    You can use this object to access all functions provided by the Turbo-BrainVoyager Exchanger Plug-in. A list of all available functions can be found in the Doxygen's reference documentation. To use this Plug-in a connection to the Turbo-BrainVoyager is necessary. A description how to establish this connection will be described in the next section.

    Connection setup

    The TBV offers different ways to access the analyzed and processed data in real-time. Using a direct TCP/IP connection a fast and reliable method to access these data. Using the TCP/IP export plug-in in TBV allows to set up a server which provides access to most of the data used in TBV. (The plug-in and its documentation can be found on the BrainVoyager home page) After setting up the server one can try to connect using the server IP and the port that was defined in the TCP/IP export plug-in. Two methods are available to connect to the server. In both cases the server must be reachable and the firewalls has to be configured to allow the connection on the defined ports.

    Automatic connection

    This command activates the automatic connect functionality to link to a TBV server if it is available and accessible. Be sure that you firewall is set up correctly and forwards the connections on port 55555. 

    bool activateAutoConnection();

    Direct connection

    This command tries to connect to the TBV server on the specified IP address and port. Be sure that you firewall is set up correctly and forwards the connections on the specified port. The function will return true, if the connection was established correctly. 

    bool connectToServer(QString sIPAddress, int port);

    Available script functionality

    After setting up the connection one can call different script TBV Exchanger Plug-in member functions, which provide access to the data from the TBV.
