Introduction

    For developers who wish to build their own StimulGL program or one of its default Plug-ins they can clone a git repository from https://github.com/svengijsen/StimulGL. This repository can be build/used/distributed under the terms of the GNU General Public License (GPL version 3), see http://www.gnu.org/licenses/gpl.html

    Build Tools

    You can build your own version of StimulGL using Microsoft Visual Studio 2012. Both solution/project files can be found inside the repository. Soon StimulGL can be build for various platforms (Windows/Linux/Mac) with 32bit or 64bit support, at this moment only Windows 32 and 64-bit platforms are supported. StimulGL development will always try to stay closely in sync with the latest version of the free libraries it makes use of like the Qt library. Only plug-ins that are build using the same version of the Qt library used for building StimulGL can be used (are compatible) with StimulGL.

    Development Environment Setup

    Setting up an Development Environment for building StimulGL can be difficult due to the fact that it uses many free and/or open source third party libraries. To make the setup and configuration easier for you we created a single custom repository containing all the source code and libraries for building and running StimulGL with Microsoft Visual Studio 2012 (with update 3). If you can install a full version of MSVS2012 and wish to setup an Development Environment that let's you build your own custom version of StimulGL or a StimulGL plug-in we advice you to contact us at sven dot gijsen at maastrichtuniversity dot nl. Collaborators who wish to help improving StimulGL or the StimulGL plug-ins are off course warmly invited!

    The StimulGL repository

    The repository contains the following directories structure:

    /Source directory:
    Here you can find all the StimulGL source code together with the Project/Solution files. Inside the /Source directory you find a /Plugin and /StimulGL folder. The /Plugin folder contains all the Plug-in projects that can be used by the StimulGL main program. The source code for the StimulGL program can be found in the /StimulGL folder.

    /Install directory:
    This directory contains all the needed files and binaries that you need for a valid installation. Some of them first need to be build by your IDE and some of them are already available in the /Install directory. A post build command can copy the build output from the IDE to this folder.

    /InstallJammer directory: This directory is the root directory for the InstallJammer project files and generated files. InstallJammer is used for building the various installers for the StimulGL application.
